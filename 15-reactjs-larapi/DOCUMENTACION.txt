## 🛠 Guía de Comandos y Funcionamiento: React + Vite

### 1. Comandos de Terminal (Instalación y Ejecución)

| Comando | ¿Para qué sirve? |
| --- | --- |
| **`npm install`** | Es el comando "maestro". Lee tu archivo `package.json` e instala todas las librerías necesarias para que el proyecto funcione. Se usa al inicio o si borras la carpeta `node_modules`. |
| **`npm install sweetalert2`** | Instala específicamente la librería de alertas. Agrega el código de SweetAlert a tu proyecto para que puedas usar el `import Swal`. |
| **`npm run dev`** | **Es el más importante.** Enciende el servidor local de desarrollo. Sin este comando, no puedes ver tu página en el navegador. |

---

### 2. Atajos del Servidor (Vite Shortcuts)

Cuando ejecutas `npm run dev`, Vite te ofrece unos atajos rápidos en la terminal (como viste en tu captura `image_9e3d1e.png`):

* **`r + enter` (Restart):** Reinicia el servidor. Útil si la página se queda congelada o no detecta un archivo nuevo.
* **`u + enter` (Show URL):** Te vuelve a mostrar la dirección (ej. `http://localhost:5173/`) por si la perdiste de vista.
* **`o + enter` (Open):** Te abre automáticamente la aplicación en tu navegador predeterminado.
* **`c + enter` (Clear):** Limpia todos los mensajes de la terminal para que la veas vacía y ordenada.
* **`q + enter` (Quit):** Apaga el servidor. La página dejará de funcionar en el navegador hasta que vuelvas a dar `npm run dev`.

### 3. Explicación de los errores y avisos iniciales

Al principio te salieron varios mensajes técnicos. Aquí te explico qué significaban esos "datos extraños":

#### A. El Error de Importación (`image_9ecc44.png`)

Vite utiliza un sistema llamado **Import Analysis**. Antes de mostrarte la página, revisa que todos los `import` tengan sentido.

> El mensaje **"Failed to resolve import 'sweetalert2'"** te estaba diciendo: *"Oye, en el código me pides usar SweetAlert, pero fui a buscarlo a la carpeta de librerías y no existe"*. Se solucionó instalándolo.

#### B. Hot Module Replacement (HMR)

Seguro viste las letras **HMR** en la terminal. Significa que Vite está vigilando tus archivos. Cada vez que das `Ctrl + S` (Guardar) en VS Code, Vite envía solo el pequeño pedazo de código que cambiaste al navegador. Por eso la página se actualiza casi instantáneamente sin recargarse por completo.

#### C. Local vs Network

* **Local:** Es la dirección que solo funciona en tu computadora.
* **Network:** Si estás conectado al mismo Wi-Fi, podrías abrir esa dirección en tu celular para ver cómo se ve tu App de mascotas en el móvil en tiempo real.

📝 Documentación Técnica: Sistema de Gestión de Mascotas (React + Laravel API)

Node.js (Entorno de ejecución para JavaScript).

npm (Gestor de paquetes de Node).

Vite (Herramienta de construcción rápida para el frontend) // Opcional.

2. Inicialización del Proyecto
Para crear la estructura base del proyecto con React y Vite, utilizamos:

npm create vite@latest my-pet-app -- --template react
cd my-pet-app
npm install

/===================================================================/
3. Instalación de Dependencias Críticas
Para que el sistema sea una SPA (Single Page Application) con comunicación segura, instalamos las siguientes librerías:

npm install react-router-dom axios sweetalert2

📋 Descripción de las Librerías:
React Router DOM: Gestiona la navegación entre vistas (/login, /pets) sin recargar el navegador.

Axios: Cliente HTTP para realizar peticiones (GET, POST, PUT, DELETE) a la API de Laravel.

SweetAlert2: Biblioteca para mostrar alertas y confirmaciones estéticas y animadas.

/===================================================================/
4. Arquitectura de Comunicación (Axios & Seguridad)
Para centralizar la seguridad, creamos una instancia de Axios con un Interceptor de Peticiones.

Comando de configuración:
Se crea un archivo src/api/axios.js para automatizar el envío del JWT (JSON Web Token):

// Este interceptor inyecta el token automáticamente en cada petición
api.interceptors.request.use((config) => {
    const token = localStorage.getItem('authToken');
    if (token) {
        config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
});

/===================================================================/
5. Estructura de Rutas y Navegación
Utilizamos el componente BrowserRouter y los Hooks de react-router-dom para el manejo de la aplicación:

useNavigate(): Para redirecciones programáticas (ej. después de un Login exitoso).

useParams(): Para capturar el ID de la mascota desde la URL en las vistas de Edición y Detalle.

Maps: Para proteger rutas y redirigir usuarios no autenticados.

/===================================================================/
6. Lógica de Componentes y Estado
Implementamos Hooks nativos de React para la reactividad del sistema:

useState: Gestiona la memoria local del componente (lista de mascotas, datos del formulario).

useEffect: Maneja efectos secundarios, como la carga inicial de datos desde la API al montar el componente PetList.

/===================================================================/
